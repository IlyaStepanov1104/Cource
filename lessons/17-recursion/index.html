Функция, которая вызывает саму себя

Что такое рекурсия?

Функция, которая в своём теле вызывает саму себя

def countdown(n):
    print(n)
    countdown(n - 1)   # вызывает саму себя
Это законный Python - ничего магического здесь нет

Задача внутри содержит задачу поменьше - и так до самой простой

countdown(n)

def countdown(n):
    if n == 0:
        print("Пуск!")
        return
    print(n)
    countdown(n - 1)
Каждый вызов делает одно простое действие и передаёт задачу дальше

Базовый случай

Когда рекурсия останавливается

Что будет без базового случая?

def countdown(n):
    print(n)
    countdown(n - 1)   # нет условия остановки!
RecursionError: maximum recursion depth exceeded

Python остановит программу сам - иначе она съела бы всю память

Структура любой рекурсии

def recursive(n):
    if <базовый случай>:    # условие остановки
        return <результат>

    # рекурсивный шаг: задача становится меньше
    return recursive(n - 1)

Базовый случай

  • Самый простой вариант
  • Отвечает сразу без вызова себя
  • Останавливает цепочку

Рекурсивный шаг

  • Делает задачу меньше
  • Приближает к базовому случаю
  • Вызывает себя с новым аргументом

factorial(n)

def factorial(n):
    if n == 0:                    # базовый случай
        return 1
    return n * factorial(n - 1)   # рекурсивный шаг
4! = 4 * 3! = 4 * 3 * 2! = 4 * 3 * 2 * 1 * 0! = 4 * 3 * 2 * 1 * 1 = 24

Трассировка factorial(4)

factorial(4)
  4 * factorial(3)
        3 * factorial(2)
              2 * factorial(1)
                    1 * factorial(0)
                              = 1     # базовый случай
factorial(0) = 1
factorial(1) = 1 * 1 = 1
factorial(2) = 2 * 1 = 2
factorial(3) = 3 * 2 = 6
factorial(4) = 4 * 6 = 24
Сначала разворачиваемся вниз до базового случая, потом считаем снизу вверх

fib(n) - каждый вызов порождает два

def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)   # два рекурсивных вызова!
fib(4)
fib(3)          fib(2)
fib(2)    fib(1)    fib(1)    fib(0)

fib(4) = 9 вызовов. fib(10) = 177. fib(50) - миллиарды. Растёт как 2^n

Линейная vs ветвящаяся рекурсия

Линейная

  • 1 рекурсивный вызов
  • factorial, countdown
  • Дерево - одна ветка
  • O(n)

Ветвящаяся

  • 2+ рекурсивных вызова
  • fibonacci
  • Дерево расширяется
  • O(2^n) - очень дорого
fib(50) наивной рекурсией - программа зависнет

Как оценить сложность рекурсии

Глубина
->
сколько уровней в дереве вызовов
Ветвление
->
сколько рекурсивных вызовов делает один шаг
Итого
->
ветвление ^ глубина = количество узлов в дереве
factorial: 1^n = O(n)    fibonacci: 2^n = O(2^n)

Рекурсия vs цикл

Рекурсия лучше

  • Деревья и графы
  • Вложенные структуры
  • Задачи "разделяй и властвуй"
  • Код выглядит как математика

Цикл лучше

  • Простая итерация
  • Нужна глубина больше 1000
  • Критична скорость и память
  • Нет естественного разбиения
