Класс Logger

import time
from colorama import Fore, Style, init
init()

class Logger:
    def log_attack(self, attacker, defender, damage):
        print(
            f"{Fore.YELLOW}{attacker.name}{Style.RESET_ALL} атакует "
            f"{Fore.RED}{defender.name}{Style.RESET_ALL} за {damage}. "
            f"HP: {defender.hp}/{defender.max_hp}"
        )
        time.sleep(0.4)

    def log_winner(self, winner):
        print(
            f"\n{Fore.GREEN}Победил: {winner.name} "
            f"HP {winner.hp}/{winner.max_hp}{Style.RESET_ALL}"
        )

Почему sleep внутри логгера

sleep в fight()

  • fight знает про задержку
  • логика разбросана
  • хочешь убрать - ищи в двух местах

sleep в logger

  • задержка - часть вывода
  • всё в одном месте
  • fight не знает про время
Каждый модуль отвечает за своё

Обновляем fight.py

from log import Logger

logger = Logger()

def fight(a, b):
    if randint(0, 1):
        a, b = b, a
    for _ in range(FIGHT_LIMIT):
        damage = a.attack(b)
        logger.log_attack(a, b, damage)
        if not b.is_alive():
            logger.log_winner(a)
            return a
        damage = b.attack(a)
        logger.log_attack(b, a, damage)
        if not a.is_alive():
            logger.log_winner(b)
            return b

Класс Item

Предметы в коде

Что описывает предмет

  • name - "Зелье здоровья"
  • effect_type - что делает: "heal" или "damage"
  • value - на сколько: 30 HP
Три поля - полное описание любого предмета

Зачем строка, а не число

Просто число

item = Item("Зелье", 30)
# 30 - это лечение? урон?

Строка-тип

item = Item("Зелье", "heal", 30)
# сразу понятно
Код читается как текст - меньше путаницы

Файл items.py

  • создаём предметы один раз как константы
  • fight.py импортирует нужные - и не знает про остальные
  • добавить новый предмет = одна строка в одном файле
Каждый файл знает только то, что ему нужно

Inventory

Рюкзак героя

Что умеет инвентарь

  • хранит список предметов
  • add_item(item) - положить в рюкзак
  • use_item(index, target) - применить и убрать
  • __str__ - показать содержимое

use_item - не просто применить

  • читаем предмет по индексу
  • применяем эффект через effect_type
  • pop(index) - убираем из списка
Без pop зелье можно выпить бесконечно

Инвентарь в Character

  • добавляем self.inventory = Inventory() в __init__
  • у каждого героя свой объект - не общий список
Теперь любой герой может держать предметы

Интерактивный бой

Игрок выбирает

Как меняется механика

Было

  • оба авто-атакуют
  • игрок только смотрит

Стало

  • игрок выбирает действие
  • враг авто-атакует в ответ
  • зелья - осознанный выбор
Каждый ход - одно действие: атака или предмет

Что видит игрок каждый ход

=== Твой ход ===
Артур: HP 55/120 | Инвентарь: Зелье здоровья
Мерлин: HP 40/70

[1] Атаковать
[2] Использовать предмет
Выбор:

Алгоритм хода игрока

1
Показываем состояние героев и инвентарь
2
Читаем input() - "1" или "2"
3
"1" - атакуем врага, "2" - выбираем предмет из инвентаря
4
Враг отвечает авто-атакой

Сбежать с боя

  • вариант "3" в меню
  • fight() возвращает None - нет победителя
  • main.py проверяет результат и реагирует
Оставим как задание - реализуй сам после урока
