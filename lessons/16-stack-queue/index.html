Очередь

Queue

Что мы уже знаем

Массив (list)

  • Элементы рядом в памяти
  • Чтение по индексу — O(1)
  • Удаление из начала — O(n)

Связный список

  • Элементы разбросаны в памяти
  • Чтение по индексу — O(n)
  • Вставка в начало — O(1)
💡 Выбор структуры данных зависит от задачи

Почему list не подходит?

Удалить последний элемент - легко:

items.pop()    # O(1) - быстро

Но очередь отдаёт первый:

items.pop(0)   # O(n) - сдвигает все элементы!
💡 В массиве элементы лежат подряд в памяти - при удалении с начала всё сдвигается

Решение: collections.deque

Двусторонняя очередь - O(1) с обоих концов

from collections import deque

q = deque()
q.append("A")      # добавить справа - O(1)
q.popleft()        # забрать слева  - O(1)
Под капотом deque - связный список: нет сдвигов при работе с концами

Стек вызовов

Call Stack

Как Python выполняет функции

Вызов
->
новый кадр кладётся на стек вызовов
Локальные vars
->
хранятся внутри кадра, исчезают вместе с ним
Возврат
->
кадр снимается, управление возвращается вниз
Python сам использует стек для управления вызовами - это не абстракция, это реальность

Что запомнить

  • Очередь - FIFO: кто первым вошёл, тот первым вышел
  • Стек - LIFO: кто последним вошёл, тот первым вышел
  • list.pop(0) это O(n) - для очереди нужен deque
  • Python сам использует стек вызовов при каждом вызове функции
