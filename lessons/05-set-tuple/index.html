Задача

Если элемент не найден - верни место вставки,
чтобы массив остался отсортированным.

Пример

arr = [2, 4, 6, 8, 10]
# ищем 9
# ответ: 3.5  (между [3] и [4])

Ещё пример

arr = [2, 4, 6, 8, 10]
# ищем 5
# ответ: 1.5  (между [1] и [2])

Решение

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return left - 0.5  # место вставки
Когда цикл заканчивается без находки - left стоит ровно на месте вставки.
Возвращаем среднее между left-1 и left - (left-0.5).

tuple

неизменяемый список

Что такое tuple

list

coords = [10, 20]
coords[0] = 99   # OK
coords.append(30)  # OK

tuple

coords = (10, 20)
coords[0] = 99   # TypeError!
coords.append(30)  # AttributeError!
tuple - это неизменяемый список.
Один раз создали - изменить нельзя.

Создание

point = (3, 7)
rgb = (255, 128, 0)
single = (42,)         # обязательна запятая!
empty = ()

# tuple() из списка
t = tuple([1, 2, 3])   # (1, 2, 3)
Индексация - как в list: point[0] -> 3
Срезы - тоже работают: rgb[1:] -> (128, 0)

Методы tuple

t = (3, 1, 4, 1, 5, 9, 1)

len(t)          # 7         - длина
t[2]            # 4         - доступ по индексу
t[1:4]          # (1, 4, 1) - срез
t.count(1)      # 3         - сколько раз встречается 1
t.index(5)      # 4         - индекс первого вхождения 5
5 in t          # True      - проверка вхождения
# стандартные функции тоже работают
min(t)          # 1
max(t)          # 9
sum(t)          # 24
sorted(t)       # [1, 1, 1, 3, 4, 5, 9]  ← возвращает list!

Распаковка (unpacking)

point = (3, 7)
x, y = point
print(x)  # 3
print(y)  # 7
# функция возвращает несколько значений
def get_hp_and_mana():
    return 100, 50       # на самом деле возвращает tuple

hp, mana = get_hp_and_mana()
print(hp, mana)  # 100 50
Когда функция "возвращает два значения" - она возвращает tuple.

Когда что использовать

tuple

  • Координаты: (x, y)
  • Цвет: (255, 0, 0)
  • Дата: (2026, 4, 3)
  • Возврат нескольких значений
  • Данные не должны меняться

list

  • Инвентарь игрока
  • Очередь врагов
  • История ходов
  • Любые данные, которые будут меняться

set

множество

Что такое set

set - это коллекция уникальных элементов без порядка.
Как математическое множество из алгебры. 
items = {3, 1, 4, 1, 5, 9, 2, 6, 5, 3}
print(items)    # {1, 2, 3, 4, 5, 6, 9}
#                  дубликаты удалены, порядок не гарантирован
Порядок не определён: {1, 2, 3} - но не set[0] -> ошибка.

Создание и операции

s = {1, 2, 3}
s = set([1, 2, 2, 3])  # из списка - {1, 2, 3}
s = set()              # пустое множество (не {}!)
3 in s          # True - проверка за O(1)
len(s)          # количество элементов

Методы set

s = {1, 2, 3}

s.add(4)            # {1, 2, 3, 4}   - добавить элемент
s.remove(2)         # {1, 3, 4}      - удалить (KeyError если нет)
s.discard(99)       # {1, 3, 4}      - удалить (без ошибки если нет)
s.pop()             # вернёт и удалит случайный элемент
s.clear()           # set()          - очистить
a = {1, 2, 3}
b = {3, 4, 5}

a.update(b)         # {1, 2, 3, 4, 5} - добавить все из b
a.copy()            # новый set с теми же элементами
a = {1, 2, 3}
b = {1, 2}

b.issubset(a)       # True  - b входит в a?  (b <= a)
a.issuperset(b)     # True  - a содержит b?  (a >= b)
a.isdisjoint({9})   # True  - нет общих элементов?

Операции над множествами

a = {1, 2, 3, 4}
b = {3, 4, 5, 6}
Операция Синтаксис Результат Смысл
Объединение a | b {1,2,3,4,5,6} всё из обоих
Пересечение a & b {3, 4} только общие
Разность a - b {1, 2} есть в a, нет в b
Симм. разность a ^ b {1,2,5,6} не в обоих сразу

Применение 1: убрать дубликаты

loot = ["меч", "щит", "меч", "зелье", "щит", "меч"]

unique = list(set(loot))
print(unique)  # ['щит', 'меч', 'зелье']
               # порядок не сохраняется!

Применение 2: быстрый поиск

list: O(n)

banned = ["admin", "root",
          "test", "guest"]

if "root" in banned:  # O(n)
    print("Запрещено")

set: O(1)

banned = {"admin", "root",
          "test", "guest"}

if "root" in banned:  # O(1)
    print("Запрещено")
При 1 000 000 элементов разница в скорости - в миллион раз.

Что нельзя класть в set

s = {1, "hello", (1, 2)}   # OK - всё hashable

s.add([1, 2, 3])    # TypeError: unhashable type: 'list'
s.add({"a": 1})     # TypeError: unhashable type: 'dict'
В set можно класть только неизменяемые объекты.
list и dict изменяемы - их нельзя.
Грубо говоря, set - это словарь, где ключ равен значению:
set.add(3) ≈≈ dict[3] = 3
Однако внутри set нет значений - только ключи. Поэтому нельзя положить list - он не может быть ключом.
Также это позволяет экономить в памяти - не нужно хранить лишние значения

frozenset

неизменяемое множество

frozenset - это set как tuple

set

s = {1, 2, 3}
s.add(4)      # OK
s.remove(1)   # OK

# В другой set - нельзя:
ss = {s}      # TypeError!

frozenset

fs = frozenset({1, 2, 3})
fs.add(4)     # AttributeError!
fs.remove(1)  # AttributeError!

# В другой set - можно:
ss = {fs}     # OK!
frozenset можно использовать как ключ в dict или хранить в другом set.
Все операции | & - ^ работают так же.

Методы frozenset

Всё, что читает - работает. Всё, что изменяет - нет.

Есть

fs = frozenset({1, 2, 3, 4})

len(fs)         # 4
2 in fs         # True
fs | {5}        # frozenset({1,2,3,4,5})
fs & {2, 3}     # frozenset({2, 3})
fs - {1}        # frozenset({2, 3, 4})
fs.issubset({1,2,3,4,5})  # True
fs.copy()       # копия

Нет

fs.add(5)       # AttributeError
fs.remove(1)    # AttributeError
fs.discard(1)   # AttributeError
fs.pop()        # AttributeError
fs.clear()      # AttributeError
fs.update({5})  # AttributeError

Python list - это динамический массив указателей.
В памяти подряд лежат не сами объекты, а ссылки на них.

Когда места не хватает - Python выделяет блок вдвое больше и копирует указатели.
Это дорого, но случается редко - поэтому append остаётся O(1) в среднем.

Python list: асимптотика

Операция Сложность Почему
lst[i] O(1) прямой адрес = начало + i
len(lst) O(1) длина хранится отдельно
lst.append(x) O(1)* * амортизированное - иногда копирует
lst.pop() O(1) просто уменьшает длину
x in lst O(n) перебирает все элементы
lst.insert(i, x) O(n) сдвигает все правее i
lst.pop(i) / del lst[i] O(n) сдвигает все правее i влево
lst.sort() O(n logn) Узнаем позже...

append: почему O(1)*

Иногда append копирует весь список - это O(n).
Но смотрим не на одну операцию, а на среднюю стоимость по серии. 
Ёмкость:  1 -> 2 -> 4 -> 8 -> 16 -> ...
Копий:       1   2   4   8   ...
После n вызовов append суммарно скопировано:
1 + 2 + 4 + ... + n/2 ≈ n
n операций - n копирований итого.
1 копирование на один append в среднем -> O(1).
Аналогия: каждый элемент при вставке кладёт в копилку 2 монеты.
Когда наступает переезд - платит из своей копилки. Монет всегда хватает.

Когда что выбирать

list
Упорядоченные данные с дублями.
Нужен доступ по индексу.
tuple
Неизменяемая группа значений.
Координаты, цвет, возврат из функции.
set
Уникальность + быстрая проверка вхождения.
Порядок не нужен.
frozenset
Множество, которое нужно хранить внутри другого set или dict.
