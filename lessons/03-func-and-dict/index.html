Функции

Зачем нужны функции?

Без функции

# Бой 1
hp = 100
damage = 30
new_hp = hp - damage
print(f"Было: {hp}, стало: {new_hp}")

# Бой 2
hp = 70
damage = 45
new_hp = hp - damage
print(f"Было: {hp}, стало: {new_hp}")

# Бой 3
hp = 35
damage = 20
new_hp = hp - damage
print(f"Было: {hp}, стало: {new_hp}")

С функцией

def take_damage(hp, damage):
    new_hp = hp - damage
    print(f"Было: {hp}, стало: {new_hp}")
    return new_hp


take_damage(100, 30)
take_damage(70, 45)
take_damage(35, 20)
Функция - это именованный блок кода, который можно вызывать сколько угодно раз

Синтаксис функции

def greet(name):                    # def, имя, аргументы
    message = f"Привет, {name}!"    # тело функции (отступ!)
    return message                  # возвращаемое значение


result = greet("Артём")   # вызов функции
print(result)             # Привет, Артём!
def add(a, b):
    return a + b


print(add(3, 5))    # 8
print(add(10, 20))  # 30

Функция без return

def show_hp(hp):
    print(f"HP: {hp}")  # просто выводит, ничего не возвращает


show_hp(100)  # HP: 100

result = show_hp(100)
print(result)
Если в функции нет return, она возвращает None.
print - это функция без return, она не возвращает строку!

Аргументы по умолчанию

def attack(damage, weapon="меч"):
    print(f"Атака {weapon} - {damage} урона")


attack(50)           # Атака меч - 50 урона
attack(80, "лук")    # Атака лук - 80 урона
attack(30, "копьё")  # Атака копьё - 30 урона
Аргументы с дефолтным значением должны быть после обязательных аргументов

Именованные аргументы

def create_character(name, hp, level):
    print(f"{name}: HP={hp}, LVL={level}")


# Позиционные - порядок важен
create_character("Артём", 100, 5)

# Именованные - порядок не важен
create_character(hp=100, name="Артём", level=5)
Именованные аргументы делают вызов функции читаемее, особенно когда параметров много

*args

*args - произвольное количество позиционных аргументов 
def total_damage(*args):
    return sum(args)


total_damage(10, 20, 30)     # 60
total_damage(5, 15, 25, 35)  # 80
def show_inventory(*items):
    for item in items:
        print(f"- {item}")


show_inventory("меч", "щит", "зелье")

**kwargs

**kwargs - произвольное количество именованных аргументов 
def create_character(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")


create_character(name="Артём", hp=100, level=5)
# name: Артём
# hp: 100
# level: 5
*args и **kwargs - это соглашение по именованию.
Главное - звёздочки: * и **

А что такое kwargs?

def create_character(**kwargs):
    print(kwargs)


create_character(name="Артём", hp=100, level=5)
# {"name": "Артём", "hp": 100, "level": 5}

Типизация (type hints)

Без типов

def add(a, b):
    return a + b

С типами

def add(a: int, b: int) -> int:
    return a + b
def greet(name: str) -> str:
    return f"Привет, {name}!"


def take_damage(hp: int, damage: int) -> int:
    return hp - damage
Python не проверяет типы при выполнении - это подсказки для IDE и разработчика

Словарь (dict)

Словарь - структура ключ -> значение. Как настоящий словарь: ищешь слово и находишь определение.

Список - по индексу

player = ["Артём", 100, 5]

player[0]  # "Артём" - кто это?
player[1]  # 100 - что это?
player[2]  # 5 - и это?

Словарь - по имени

player = {
    "name": "Артём",
    "hp": 100,
    "level": 5,
}

player["name"]   # "Артём"
player["hp"]     # 100
Словарь читается как текст - сразу понятно, что "hp" это здоровье

Работа со словарём

player = {
    "name": "Артём",
    "hp": 100,
    "level": 5,
    "inventory": ["меч", "щит"],
}

player["hp"]          # 100 - получить значение
player["mana"] = 50   # добавить новый ключ
player["hp"] = 80     # перезаписать значение
"hp" in player     # True - проверить наличие ключа
"mana" in player   # False (если не добавляли)

del player["mana"]  # удалить ключ

Получение значения по ключу

Через []

player = {"hp": 100}

player["hp"]    # 100
player["mana"]  # KeyError!

Через .get()

player = {"hp": 100}

player.get("hp")       # 100
player.get("mana")     # None
player.get("mana", 0)  # 0
Используй .get(), когда ключ может отсутствовать

Методы словаря

player = {"name": "Артём", "hp": 100, "level": 5}

player.keys()    # dict_keys(["name", "hp", "level"])
player.values()  # dict_values(["Артём", 100, 5])
player.items()   # dict_items([("name", "Артём"), ...])
stats = {"attack": 30, "defense": 15}
player.update(stats)  # добавить несколько ключей сразу

# player теперь содержит: name, hp, level, attack, defense

Словарь в циклах

player = {"name": "Артём", "hp": 100, "level": 5}

for key in player:  # перебор ключей
    print(key)
for key, value in player.items():  # ключ и значение
    print(f"{key}: {value}")

# name: Артём
# hp: 100
# level: 5
Чаще всего нужен .items() - он даёт и ключ, и значение сразу

Что запомнить

  • def + return - функция принимает аргументы и возвращает результат
  • Аргументы по умолчанию и именованные аргументы делают вызов читаемее
  • Словарь - хранение данных по ключу, а не по индексу
  • .get(key, default) - безопасное получение значения
  • .items() - перебор словаря в цикле
  • Словарь-счётчик - очень частый паттерн в задачах
