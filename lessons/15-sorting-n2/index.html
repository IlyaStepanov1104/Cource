Хуже квадратичных

Как сортировать совсем плохо

Мысленный эксперимент

  • есть неотсортированный массив
  • что если просто перемешивать его случайно?
  • рано или поздно нужный порядок выпадет
звучит безумно - но это настоящий алгоритм с названием

Bogosort: идея

1
Проверь - отсортирован ли массив?
2
Если нет - перемешай случайно
3
Повтори с шага 1

Bogosort: код

import random
def is_sorted(arr):
    for i in range(len(arr) - 1):
        if arr[i] > arr[i + 1]:
            return False
    return True

def bogosort(arr):
    while not is_sorted(arr):
        random.shuffle(arr)
  • самый короткий код сортировки
  • arr != sorted(arr) - проверяем, есть ли нужный порядок
  • shuffle - полная случайная перестановка

Bogosort: сложность

  • n элементов имеют n! возможных порядков
  • в среднем нужно n! попыток, каждая стоит O(n)
  • итого: O(n * n!)

Bubble Sort

Пузырьковая сортировка

Идея

  • идём по массиву слева направо
  • сравниваем каждых двух соседей
  • если левый больше правого - меняем местами
после каждого прохода наибольший элемент оказывается в конце

Bubble Sort в действии

Разберем на примере: [5, 3, 8, 1]

Шаги прохода 1

  • [5, 3, 8, 1] - 5 > 3, своп → [3, 5, 8, 1]
  • [3, 5, 8, 1] - 5 < 8, нет свопа
  • [3, 5, 8, 1] - 8 > 1, своп → [3, 5, 1, 8]
8 "всплыл" на своё место - как пузырёк

Шаги прохода 2

  • [3, 5, 1, 8] - 3 < 5, нет свопа
  • [3, 5, 1, 8] - 5 > 1, своп → [3, 1, 5, 8]
5 "всплыл" на своё место - как пузырёк

Шаги прохода 2

  • [3, 1, 5, 8] - 3 > 1, своп → [1, 3, 5, 8]
1 и 3 "всплыли" на своё место - сортировка закончилась

Сложность Bubble Sort

Время

O(n²) обычно

O(n) с флагом, если уже отсортирован

Память

O(1)

сортируем на месте, доп. памяти нет

Selection Sort

Сортировка выбором

Идея

  • делим массив на два куска: отсортированный слева и остаток справа
  • ищем минимум в неотсортированной части
  • ставим его на первое место неотсортированной части
  • граница сдвигается на один вправо

Selection Sort в действии

Пример: [5, 3, 8, 1]

  • мин в [5,3,8,1] = 1, своп 5↔1: [1, 3, 8, 5]
  • мин в [3,8,5] = 3, уже стоит: [1, 3, 8, 5]
  • мин в [8,5] = 5, своп 8↔5: [1, 3, 5, 8]
  • один элемент остался - готово

Главная особенность

Bubble Sort

до O(n²) свопов

свопает при каждом сравнении

Selection Sort

ровно O(n) свопов

один своп в конце каждого прохода

важно, когда своп дорогой - например, запись на диск или SSD

Сложность Selection Sort

Время

O(n²) всегда

нет оптимизации для уже отсортированного

Свопов

O(n)

лучший результат среди трёх алгоритмов

Insertion Sort

Сортировка вставками

Идея

  • как раскладываем карты в руке
  • берём следующую карту из колоды
  • сдвигаем большие карты вправо, пока не найдём место
  • вставляем карту на нужное место

Insertion Sort в действии

Пример: [5, 3, 8, 1]

  • берём 3: сдвигаем 5 → [3, 5, 8, 1]
  • берём 8: 5 < 8, не сдвигаем → [3, 5, 8, 1]
  • берём 1: сдвигаем 8, 5, 3 → [1, 3, 5, 8]

Лучший случай: O(n)

  • если массив уже отсортирован - сдвигать ничего не нужно
  • внутренний цикл не выполняется ни разу
  • n элементов проверили, 0 сдвигов - O(n)
лучший выбор, когда данные добавляются по одному в почти отсортированный список

Сложность Insertion Sort

Худший случай

O(n²)

массив отсортирован в обратном порядке

Лучший случай

O(n)

массив уже отсортирован

Сравнение

Когда что выбирать

Три алгоритма рядом

Алгоритм Время (худший) Время (лучший) Свопов
Bubble Sort O(n²) O(n) с флагом O(n²)
Selection Sort O(n²) O(n²) O(n)
Insertion Sort O(n²) O(n) O(n²)

Когда что использовать

  • Insertion Sort - данные почти отсортированы или приходят по одному
  • Selection Sort - минимум записей на диск (swap дорогой)
  • Bubble Sort - учебные цели, самый наглядный алгоритм
в рабочем коде - всегда list.sort() или sorted()

Почему все три - O(n²)?

  • для n элементов нужно ~n операций сравнения
  • каждое сравнение само по себе требует обойти ~n элементов
  • n * n = n²
  • выйти за O(n²) можно только через разделяй и властвуй

Следующий шаг

O(n²) сортировки

один большой массив
обрабатываем целиком

O(n log n) сортировки

делим пополам рекурсивно

Что запомнить

  • Bubble Sort - сравниваем соседей, большие всплывают вверх
  • Selection Sort - ищем минимум, ставим на место; O(n) свопов
  • Insertion Sort - вставляем как карту в руку; O(n) на почти отсортированном
  • все три - O(n²) в худшем случае из-за двух вложенных циклов
  • в реальном коде - sorted()
