Конструктор

Конструктор - это специальный метод, который Python вызывает автоматически при создании нового объекта класса. 
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

rex = Dog("Рекс", 3)   # здесь вызывается __init__
print(rex.name)  # Рекс
print(rex.age)   # 3
__init__ - это конструктор
В конструкторе задаём начальные свойства объекта

Что такое self?

self - это сам объект, который создаётся. 
rex = Dog("Рекс", 3)
# Python подставляет:
# Dog.__init__(rex, "Рекс", 3)
#              ^^^^
#              это self
Первым параметром метода всегда идёт self

self.name = name - что это?

def __init__(self, name, age):
    self.name = name   # атрибут объекта = параметр -> rex.name = "Рекс"
    self.age = age
Слева: self.name - атрибут, куда сохраняем
Справа: name - параметр, что сохраняем

Пример: персонаж игры

    def __init__(self, name, hp, damage):

hero = Character("Герой", 100, 15)
goblin = Character("Гоблин", 30, 5)

print(hero.name, hero.hp)      # Герой 100

Работает так же, как в обычных функциях

Атрибуты - у каждого свои

hero = Character("Герой", 100, 15)

Изменения одного объекта не влияют на другие

Свойства класса

Атрибуты можно задать прямо в теле класса - они будут общими значениями по умолчанию.

    health = 1000
    armour = 30

    def __init__(self, name):

h1 = Hero("Артас")
h2 = Hero("Иллидан")

print(h1.health, h1.armour)  # 1000 30
print(h2.health, h2.armour)  # 1000 30

Свойства класса vs объекта

class Hero:
    health = 1000       # свойство КЛАССА
    armour = 30

    def __init__(self, name):
        self.name = name  # свойство ОБЪЕКТА

Методы

Поведение объекта

Метод = функция внутри класса

class Dog:
    def __init__(self, name):
        self.name = name

    def bark(self):
        print(f"{self.name}: Гав!")

rex = Dog("Рекс")
rex.bark()  # Рекс: Гав!
Метод всегда получает self - сам объект

Метод меняет состояние

class Character:
    def __init__(self, name, hp):
        self.name = name
        self.hp = hp

    def take_damage(self, amount):
        self.hp -= amount
        print(f"{self.name}: -{amount} HP")

hero = Character("Герой", 100)
hero.take_damage(15)
print(hero.hp)  # 85

Метод может возвращать значение

class Character:
    def __init__(self, name, hp):
        self.name = name
        self.hp = hp

    def is_alive(self):
        return self.hp > 0

hero = Character("Герой", 0)
print(hero.is_alive())  # False
Методы, возвращающие bool, часто называют is_* или has_*

Методы взаимодействуют

class Character:
    def __init__(self, name, hp, damage):
        self.name = name
        self.hp = hp
        self.damage = damage

    def take_damage(self, amount):
        self.hp -= amount

    def attack(self, target):
        print(f"{self.name} бьёт {target.name}")
        target.take_damage(self.damage)

В действии

hero = Character("Герой", 100, 15)
goblin = Character("Гоблин", 30, 5)

hero.attack(goblin)
# Герой бьёт Гоблин

print(goblin.hp)  # 15
goblin.attack(hero)
print(hero.hp)    # 95
💡 Код читается почти как русский текст

Метод vs функция

Функция

attack(hero, goblin)
take_damage(hero, 10)

Метод

hero.attack(goblin)
hero.take_damage(10)
Метод привязан к объекту - это делает код яснее

Соглашения Python

Префикс Смысл
name Public - используй свободно
_name Protected - «не трогай снаружи»
__name Private - Python скроет имя
В Python это договорённость, а не запрет

Как работает __

class Account:
    def __init__(self):
        self.__password = "secret"

a = Account()
print(a.__password)
# AttributeError: 'Account' object has no attribute '__password'
Python переименовал атрибут при компиляции

Что сделал Python

class Account:
    def __init__(self):
        self.__password = "secret"

a = Account()
print(a.__dict__)
# {'_Account__password': 'secret'}

print(a._Account__password)  # "secret"
__password превратился в _Account__password
Достать можно, но случайно - уже не получится

Метод вместо прямого доступа

class BankAccount:
    def __init__(self, balance):
        self._balance = balance

    def deposit(self, amount):
        if amount <= 0:
            raise ValueError("Сумма должна быть > 0")
        self._balance += amount

    def withdraw(self, amount):
        if amount > self._balance:
            raise ValueError("Недостаточно средств")
        self._balance -= amount

Зачем это?

✅
Валидация - баланс не уйдёт в минус
🛡️
Защита - никто не изменит данные случайно
🔧
Гибкость - можно поменять реализацию, не ломая чужой код

Магические методы

Методы с __двумя__ подчёркиваниями - Python вызывает их сам в нужный момент.
Один ты уже знаешь - __init__ вызывается при создании объекта

__str__

Вызывается, когда объект превращают в строку: print(obj) или str(obj). 
class Character:
    def __init__(self, name, hp):
        self.name = name
        self.hp = hp

    def __str__(self):
        return f"{self.name} (HP: {self.hp})"

hero = Character("Герой", 100)
print(hero)  # Герой (HP: 100)

__int__

Вызывается при int(obj) - когда объект превращают в число. 
class Money:
    def __init__(self, amount):
        self.amount = amount

    def __int__(self):
        return self.amount

m = Money(1500)
print(int(m))  # 1500

__eq__

Вызывается при a == b. Без него Python сравнивает адреса в памяти. 
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

print(Point(1, 2) == Point(1, 2))  # True

__lt__ и __gt__

a < b, a > b. Достаточно реализовать один - и заработает сортировка. 
class Student:
    def __init__(self, name, grade):
        self.name = name
        self.grade = grade

    def __lt__(self, other):
        return self.grade < other.grade

students = [Student("Иван", 4), Student("Мария", 5)]
students.sort()  # использует __lt__

__add__, __sub__, __mul__

Арифметика: a + b, a - b, a * b. Можно определить для своих классов. 
class Money:
    def __init__(self, amount):
        self.amount = amount

    def __add__(self, other):
        return Money(self.amount + other.amount)

m1 = Money(100)
m2 = Money(250)
print((m1 + m2).amount)  # 350

__len__ и __getitem__

__len__ - для len(obj). __getitem__ - для obj[i]. 
class Team:
    def __init__(self, players):
        self.players = players

    def __len__(self):
        return len(self.players)

    def __getitem__(self, i):
        return self.players[i]

team = Team(["Иван", "Мария", "Пётр"])
print(len(team))   # 3
print(team[0])     # Иван

__contains__

Вызывается оператором in: x in obj. 
class Team:
    def __init__(self, players):
        self.players = players

    def __contains__(self, name):
        return name in self.players

team = Team(["Иван", "Мария"])
print("Иван" in team)   # True
print("Пётр" in team)   # False

__bool__

Вызывается в if obj: - определяет, что считать «пустым». 
class Inventory:
    def __init__(self, items):
        self.items = items

    def __bool__(self):
        return len(self.items) > 0

bag = Inventory([])
if bag:
    print("Что-то есть")
else:
    print("Пусто")  # напечатает это

__call__

Вызывается при obj() - объект начинает работать как функция. 
class Multiplier:
    def __init__(self, factor):
        self.factor = factor

    def __call__(self, x):
        return x * self.factor

double = Multiplier(2)
print(double(5))   # 10
print(double(7))   # 14

Шпаргалка: магические методы

Метод Когда вызывается
__init__ создание объекта: Hero()
__str__ print(obj), str(obj)
__int__ int(obj)
__len__ len(obj)
__getitem__ obj[i]
__eq__ сравнение a == b
__lt__, __gt__ a < b, a > b, сортировка
__add__, __sub__, __mul__ a + b, a - b, a * b
__contains__ x in obj
__bool__ if obj: - что считать «пустым»
__call__ obj() - объект как функция
Реализуй - и твой класс ведёт себя как родной Python-объект

Что запомнить

  • class - чертёж, объект - экземпляр
  • __init__ создаёт объект, self - сам объект
  • Атрибуты: self.x = x, у каждого объекта свои
  • Методы - функции с self, работают с данными объекта
  • _name - protected, __name - private
  • магические методы - для определения поведения объектов
