Задача Two Sum

Дан список чисел и цель. Найти индексы двух чисел, которые в сумме дают цель.

Вход

nums = [2, 7, 11, 15]
target = 9

Выход

[0, 1]
# nums[0] + nums[1] = 2 + 7 = 9

Два решения

Словарь

def two_sum_dict(nums, target):
    seen = {}
    for i, num in nums.items():
        diff = target - num
        if diff in seen:
            return [seen[diff], i]
        seen[num] = i
    return []

Двойной цикл

def two_sum_cycle(nums, target):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
Оба дают правильный ответ. Какой лучше?

Зачем усложнять?

Двойной цикл - проще читать, проще понять.
Словарь - хитрее. Нужно помнить про diff, про seen.
Есть ли реальная разница - или это просто стиль?

Измерим разницу

Как измерить время выполнения

import time

start = time.perf_counter()

# ... твой код ...

elapsed = time.perf_counter() - start
print(f"Время: {elapsed:.4f} сек")
perf_counter() возвращает текущее время в секундах.
Разница до и после - время работы кода.

Проблема: время ненадёжно

  • На твоём ноутбуке одно время, на моём - другое
  • Запустить два раза - получишь два разных результата
  • Нельзя измерить до запуска - а хочется знать заранее
Нужен способ сравнивать алгоритмы независимо от железа

Считаем операции

Формулы

Двойной цикл: перебирает все пары i, j где j > i 
n = 4:   (0,1), (0,2), (0,3), (1,2), (1,3), (2,3)   = 6 пар
n = 10:  45 пар
n = 100: 4950 пар
n * (n-1) / 2 итераций цикла
Словарь: один проход - n итераций цикла

Big O

нотация сложности

Что такое Big O

Big O описывает, как растёт число операций при увеличении n.
Мы берём только самое быстро растущее слагаемое и отбрасываем константы.

Формула

n² + 100n + 5

Big O

O(n²)
3n + 42
O(n)
При больших n константы и мелкие слагаемые перестают иметь значение

O(n²) - вложенный цикл

for i in range(n):
    for j in range(n):
        # n * n = n² операций
        ...
n=10 → порядка 100 операций
n=100 → порядка 10 000 операций
n=1000 → порядка 1 000 000 операций
Увеличиваем n в 10 раз - операций становится в 100 раз больше

O(n) - один цикл

for i in range(n):
    # n операций
    ...
n=10 → порядка 10 операций
n=100 → порядка 100 операций
n=1000 → порядка 1000 операций
Увеличиваем n в 10 раз - операций тоже в 10 раз больше

O(1) - константа

seen = {2: 0, 7: 1, 11: 2}

# Поиск в словаре - всегда одна операция
result = seen.get(7)  # мгновенно, неважно какой размер
Число операций не зависит от n.
Именно поэтому поиск в словаре такой быстрый.

O(log n) - логарифм

На каждом шаге задача уменьшается вдвое.
n=16 → 4 шага
n=1000 → 10 шагов
n=1 000 000 → 20 шагов

Сложность по памяти

Алгоритмы потребляют не только время, но и память.
Обозначается так же: O(1), O(n), O(n²)...

O(1) памяти

Несколько переменных. Не растёт с n.

j = 1

Словарь/список растут вместе с n.

# размер seen растёт

Алгоритм бинарного поиска

Обязательное условие: массив должен быть отсортирован

Насколько быстрее?

n                Линейный         Бинарный
---------------  ---------------  ---------------
10               10               4
100              100              7
1000             1000             10
1 000 000        1 000 000        20

Теория:
n = 1 000 000 000      ->  бинарный: 30 шагов
n = 1 000 000 000 000  ->  бинарный: 60 шагов
Миллиард элементов - 30 шагов. Это сила O(log n).

Что запомнить

  • Big O описывает рост числа операций - независимо от железа
  • O(1) - константа, не зависит от n
  • O(log n) - логарифм, делим пополам на каждом шаге (бинарный поиск)
  • O(n) - линейный, один проход (for-цикл)
  • O(n²) - квадратичный, вложенный цикл
  • Кроме времени - ещё и память
