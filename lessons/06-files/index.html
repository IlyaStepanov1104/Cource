Работа с файлами

Сохраняем и загружаем данные

Что такое файл?

Файл - это последовательность байтов (данные), хранящихся на диске. 
C:\Users\Artom\documents\data.txt    # Windows
/home/artom/documents/data.txt       # Linux/Mac

.txt

Текстовый файл

.json

Структурированные данные

.csv

Таблица (Excel)

.py

Код Python

Функция open()

f = open("файл.txt", "r")  # открыть файл

Режимы доступа:

"r" Чтение (по умолчанию)
"w" Запись (перезапишет файл)
"a" Добавление в конец файла
"r+" Чтение и запись (файл должен существовать)
"w+" Запись и чтение (создаст/перезапишет файл)
"a+" Добавление и чтение
r+ vs w+: r+ ждёт существующего файла, w+ создаст новый или перезапишет старый

Открытие и закрытие файла

f = open("data.txt", "r")  # открыли файл

# работаем с файлом

f.close()  # закрыли файл
Каждый open() должен иметь соответствующий close()

Почему закрывать файлы?

1
Блокировка - другие программы не могут менять файл
2
Потеря данных - буфер может не сбросить данные на диск
3
Ресурсы - каждый файл занимает память
4
"Too many open files" - ошибка при открытии 1000+ файлов
⚠️ Всегда закрывай файл после использования!

Context manager: with

❌ Плохо

f = open("data.txt")
data = f.read()
f.close()  # забыл?

✅ Хорошо

with open("data.txt") as f:
    data = f.read()
# файл закроется автоматически!
with закроет файл всегда - даже если будет ошибка!

Методы чтения файла

with open("data.txt") as f:
    f.read()       # весь файл строкой
    f.readline()   # одна строка
    f.readlines()  # все строки списком

    for line in f:  # итерация по строкам
        print(line)

Методы записи в файл

with open("data.txt", "w") as f:
    f.write("текст")           # записать строку
    f.write(str(42) + "\n")    # число -> строка

    lines = ["строка1\n", "строка2\n"]
    f.writelines(lines)        # записать список строк

    print("Привет", file=f)    # print в файл
write() не добавляет \n автоматически, а print() добавляет

JSON и CSV

Структурированные данные

Что такое JSON?

JSON = JavaScript Object Notation - универсальный текстовый формат для данных. 
{
  "name": "Артём",
  "level": 5,
  "items": ["меч", "щит", "зелье"]
}
JSON работает во всех языках программирования

Соответствие типов

Python JSON
dict {}
list []
str "string"
int / float 123 / 3.14
True / False true / false
None null

Модуль json

import json

# Чтение из файла
with open("player.json") as f:
    player = json.load(f)  # dict

# Запись в файл
with open("player.json", "w") as f:
    json.dump(player, f, indent=2)
indent=2 делает JSON красивым и читаемым

Что такое CSV?

CSV = Comma-Separated Values - табличный формат (как Excel). 
name,score,level
Иван,1500,5
Мария,1200,4
Петя,900,3
CSV хорошо открывается в Excel и Google Sheets

Модуль csv

import csv

# Запись
with open("scores.csv", "w") as f:
    writer = csv.writer(f)
    writer.writerow(["name", "score"])
    writer.writerow(["Иван", 1500])

# Чтение
with open("scores.csv") as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)
        # ['name', 'score']
        # ['Иван', '1500']

JSON vs CSV - Когда использовать?

JSON

✓ Сложные структуры
✓ Вложенные данные
✓ Конфиги

CSV

✓ Таблицы
✓ Рекорды, логи
✓ Экспорт в Excel

Исключения

Обработка ошибок

Синтаксические vs Runtime ошибки

Синтаксис ❌

Python не может парсить код

if x > 5  # нет :

Runtime ❌

Программа падает при выполнении

int("abc")  # ValueError
Синтаксические ошибки видно при запуске, а Runtime ошибки могут возникать в процессе работы программы

Try-except

try:
    age = int(input("Возраст: "))
    print(f"Тебе {age} лет")
except ValueError:
    print("Ошибка! Введи число")
Код в try выполняется нормально.
Если ошибка -> прыгаем на except.
Программа не падает!

Типы исключений

FileNotFoundError   # файл не найден
ValueError          # неправильный формат данных
KeyError            # ключа нет в словаре
ZeroDivisionError   # деление на 0
IndexError          # индекс вне границ списка
TypeError           # неправильный тип данных
except FileNotFoundError:
    print("Файл не существует")
except ValueError:
    print("Неправильный формат")

Ловим все ошибки

try:
    # какой-то код
    x = int(input())
    y = 10 / x
except Exception:
    print("Что-то пошло не так!")
except Exception: ловит все ошибки
Но лучше ловить конкретные ошибки!

Finally - Очистка ресурсов

try:
    f = open("data.txt")
    data = f.read()
except FileNotFoundError:
    print("Файл не найден")
finally:
    f.close()  # выполнится ВСЕГДА
finally выполняется всегда - даже если была ошибка
Используется для закрытия файлов, соединений с БД и тд

Raise - Выброс исключения

age = int(input("Возраст: "))

if age < 0:
    raise ValueError("Возраст не может быть отрицательным!")

print(f"Тебе {age} лет")
raise выбрасывает исключение для валидации данных

Что запомнить

  • Всегда использовать with для файлов
  • JSON для сложных структур, CSV для таблиц
  • try-except для перехвата ошибок
  • finally для очистки ресурсов
  • raise для валидации данных
