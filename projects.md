# Проекты курса

## Проекты по этапам

### Python-проект (уроки 9-10)

К этому моменту ученик знает: переменные, строки, функции, словари, set/tuple, файлы, ООП, модули, Git.

1. **Консольная RPG** - персонаж с характеристиками (словарь), инвентарь (список), консольный бой через функции, сохранение в JSON. Лучший вариант для сквозного проекта.
2. **Менеджер задач** - создать задачу, отметить выполненной, удалить, сохранить в JSON. Близко к реальной разработке, хорошо для портфолио.
3. **Виртуальный питомец** - характеристики (голод, счастье, здоровье), действия (покормить, поиграть), эволюция при достижении порогов, сохранение состояния.
4. **Викторина** - вопросы и ответы из JSON-файла, подсчёт очков, таблица рекордов в файле, несколько категорий вопросов.
5. **Текстовый квест** - сюжетные развилки через словари, инвентарь, несколько концовок в зависимости от выборов.
6. **Библиотека** - каталог книг, поиск по автору/жанру, выдача и возврат, история чтения, сохранение в JSON.

---

### Pygame мини-игра (уроки 14-15)

К этому моменту ученик знает: Python полностью + Big O + бинарный поиск + массивы + рекурсию.

1. **Аркада** - персонаж уворачивается от падающих объектов, скорость растёт, таблица рекордов. Просто реализуется, хорошо выглядит.
2. **Змейка** - классика, хорошо демонстрирует работу со списком координат.
3. **Простой платформер** - персонаж, гравитация, один уровень, монеты или враги.
4. **Space Invaders** - ряды врагов, стрельба, щиты, нарастающая сложность.
5. **Pong** - двое игроков или один против AI, хорошо показывает логику столкновений.
6. **Кликер** - появляются цели, нужно кликнуть за время, счётчик промахов, таймер.

---

### Проект с SQLite (урок 21)

К этому моменту ученик знает: всё выше + стек/очередь + хеш-таблицы + SQL.

1. **Таблица рекордов к Pygame-игре** - история партий (дата, счёт, длительность), топ-10, фильтрация по дате. Органично вырастает из Pygame-проекта.
2. **Личный дневник** - записи с датой и тегами, полнотекстовый поиск, статистика по настроению.
3. **Трекер привычек** - ежедневные отметки, стрики, процент выполнения за месяц, графики в консоли.
4. **Каталог медиа** - фильмы/игры/книги, оценки, статусы (просмотрено/в процессе/хочу), рекомендации по жанру.
5. **Трекер расходов** - категории, суммы, дата, месячные отчёты, поиск по категории.
6. **Менеджер контактов** - имена, телефоны, группы, история взаимодействий, поиск.

---

### API мини-проект (урок 27)

К этому моменту ученик знает: всё выше + сортировки + Divide and Conquer.

1. **Прогноз погоды** - текущая погода и прогноз на 5 дней для любого города, красивый вывод в консоли.
2. **Новостной агрегатор** - последние новости по ключевому слову, сохранение интересных в SQLite.
3. **Курсы валют** - конвертер с актуальными данными, история курса, визуализация в консоли.
4. **GitHub-профиль** - статистика репозиториев пользователя, топ по звёздам, языки программирования.
5. **Случайный квест** - генератор квестов/событий для RPG-игры на основе внешнего API (Open Trivia, и тд).
6. **Переводчик** - перевод текста с автоопределением языка, история переводов в SQLite.

---

### Финальный проект (после урока 37)

Зависит от выбранной специализации. Общие варианты, которые можно адаптировать под любое направление:

1. **RPG с полным фичесетом** - если шли по сквозному проекту: Pygame + SQL + API + граф-карта + DP-инвентарь.
2. **Веб-приложение на Flask** - если бэкенд: REST API, БД, аутентификация, деплой.
3. **Полноценная игра на Pygame** - если игры: несколько уровней, меню, звук, сохранения.
4. **Дашборд с данными** - если данные/ML: парсинг данных, анализ, визуализация, простая модель.
5. **Телеграм-бот** - если хочет продолжить опыт с ботом: команды, БД, интеграция с API.
6. **Клон известного сервиса** - упрощённый аналог (заметки, трекер задач, агрегатор новостей) с полным стеком.

## Сквозные проекты на весь курс

Идея: один проект с первого занятия, который получает новые фичи после каждой темы.

---

### 1. RPG-игра

**Почему хорошо:** органично вмещает все темы, мотивирует ученика, видимый прогресс.

| Тема | Что добавляем |
| --- | --- |
| Функции и словари | Персонаж (словарь), функции `attack()`, `heal()`, `show_stats()` |
| set/tuple | Уникальные эффекты (set), координаты (tuple) |
| Файлы | Сохранение/загрузка персонажа в JSON |
| ООП | Классы `Character`, `Enemy`, `Item`; наследование `Warrior`, `Mage` |
| Модули + Git | Разбить на `player.py`, `enemy.py`, `items.py`; первый коммит |
| Python-проект | Завершить консольную версию с полным игровым циклом |
| Pygame | Переписать на визуальный интерфейс: окно, спрайты, анимация |
| Хеш-таблицы | Быстрый поиск предметов по ID в инвентаре |
| SQL | Таблица рекордов, сохранение прогресса в БД |
| Сортировки | Сортировка инвентаря по урону, весу, редкости |
| API | Случайные квесты или монстры из внешнего API |
| Деревья | Дерево диалогов и квестов |
| Графы + BFS | Карта мира как граф, поиск пути между локациями |
| Дейкстра | Оптимальный маршрут, умный AI врагов |
| DP | Задача о рюкзаке для инвентаря: максимизировать урон при ограниченном весе |

---

### 2. Симулятор города

**Почему хорошо:** стратегический жанр, хорошо ложится на алгоритмы управления ресурсами.

| Тема | Что добавляем |
| --- | --- |
| Функции и словари | Здания и ресурсы как словари, функции `build()`, `collect()` |
| Файлы | Сохранение состояния города |
| ООП | Классы `Building`, `Citizen`, `Resource` |
| Python-проект | Консольный симулятор с несколькими раундами |
| Pygame | Визуальная карта с кликабельными зданиями |
| SQL | Хранение истории строительства и статистики |
| API | Внешние события (погода, новости) влияют на ресурсы |
| Графы + BFS | Дороги между районами, логистика ресурсов |
| Дейкстра | Оптимальные маршруты для доставки |
| Жадные | Оптимальное распределение бюджета |
| DP | Максимизация счастья граждан при ограниченном бюджете |

---

### 3. Социальная сеть

**Почему хорошо:** близко к реальным проектам, граф друзей - естественная структура.

| Тема | Что добавляем |
| --- | --- |
| Функции и словари | Профиль пользователя, функции `post()`, `like()` |
| Файлы | Хранение постов в JSON |
| ООП | Классы `User`, `Post`, `Comment` |
| Python-проект | Консольная лента с постами |
| SQL | Полноценная БД: пользователи, посты, лайки, подписки |
| API | Авторизация через внешний сервис, публикация в соцсети |
| Хеш-таблицы | Быстрый поиск пользователя по username |
| Сортировки | Сортировка ленты по дате, популярности |
| Графы + BFS | Граф подписок, поиск общих друзей |
| DP | Алгоритм ленты: максимизировать вовлечённость |

### 4. Маркетплейс

**Почему хорошо:** реальная бизнес-задача, все алгоритмы применяются естественно.

| Тема | Что добавляем |

| Функции и словари | Товар (словарь), корзина, функции `add_to_cart()`, `checkout()` |

### 5. Менеджер задач

**Почему хорошо:** самая близкая к реальной разработке тема, отличное портфолио для поступления.

| Тема | Что добавляем |
| --- | --- |
| Функции и словари | Задача (словарь), функции `add()`, `done()`, `delete()` |
| Файлы | Сохранение задач в JSON |
| ООП | Классы `Task`, `Project`, `Tag` |
| Python-проект | Консольное приложение с фильтрами и приоритетами |
| SQL | БД: задачи, проекты, теги, история изменений |
| API | Уведомления, синхронизация с календарём |
| Хеш-таблицы | Быстрый поиск задачи по ID или тегу |
| Сортировки | По приоритету, дедлайну, проекту |
| Деревья | Иерархия задач: эпик -> история -> подзадача |
| Графы | Зависимости между задачами |
| Топологическая сортировка | Порядок выполнения с учётом зависимостей |
| DP | Оптимальное планирование спринта при ограниченных ресурсах |
